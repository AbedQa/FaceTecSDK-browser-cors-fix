‚ùå The Problem

CORS (Cross-Origin Resource Sharing) is a browser security feature that blocks requests from one domain (your app) to another domain (Orange APIs) unless the server explicitly allows it.

Orange API server (`https://xpapis.orange.jo`) does NOT send CORS headers, so browsers block the requests.

Error: Access to fetch at 'https://xpapis.orange.jo/xyz/sso/v1/dev/keycloak/user/login' from origin 'https://orange-jood.web.app' has been blocked by CORS policy: Response to preflight request doesn't pass access control check: No 'Access-Control-Allow-Origin' header is present on the requested resource.

‚ö†Ô∏è Important

You CANNOT fix CORS by adding headers to your client code!

CORS headers must be sent by the server (Orange API), not by your browser.

Adding `Access-Control-Allow-Origin` to your request headers does nothing.

Solution 1: Browser Extension (Easiest)

Step 1: Install "CORS Unblock" extension

Chrome: CORS Unblock

Firefox: CORS Everywhere

Step 2: Enable the extension (click the extension icon)

Step 3: Visit https://orange-jood.web.app and test!

Solution 2: Local Development

Install live-server with CORS disabled:

npm install -g live-server live-server --cors .

Then visit: http://localhost:8080/sample-apps/sample-app-js/
